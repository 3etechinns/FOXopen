Bang Handler Help

!$(BANGHANDLER)
Processes the bang handler but won't serialise tracks to DB, unless an exception is thrown
!ABOUT
Shows information about FOX and third party libraries used
!BUSYRESET
Resets the foreground busy status of the job agent and displays "BUSYRESET Success. BUSY=[true|false]" depending on whether the job agent is foreground is now busy or not
!COMPOSITE-IMAGE?images=img1;img2[&app_mnem=AppMnem]
Combines a list of images on top of each other to create a composite image
!HELP
This page, or if logged in it'll show other commands
!LOGIN
Displays a login form. Logging in allows use of the restricted bang handlers
!SHOWBUSY
Displays "BUSY=[true|false]" depending on whether the job agent is foreground busy or not
!SHOWSTATS
Displays the stats DOM
!STATUS-IMAGE[/img]/(Literal String)&(Literal String)&(Literal String)[&(Literal String)]
Returns the status of the image matching the given parameters, including a link to the image if the processing is done
!VALIDATE/ModuleWildcard[/AppMnem]
Checks wildcarded modules for errors
!CONFIGURE
Configure engine boot settings
!DOCSTART | !DOCSTOP
Starts and stops the document generation job
!EM
Re-generate the local engine mirror files
!STATUS
Shows the status of core engine components and currently loaded apps
!FLUSH
Flush the component cache, clearing cached modules/images, refreshing the engine mirror and app error pages and also flushing the FOP cache
!FLUSHDOMCACHE
Clears the DOM and XPath caches for the app server.
!FLUSHINTERNAL
Clears the internal component cache for the app server.
!FLUSHMAPSET
Clears the mapset cache for each app mnemonic on the app server.
!FLUSHTHREAD
Clears the XThread cache for the app server.
!FLUSHUPLOAD
Clears the UploadInfo cache for the app server. Use with caution - this will cancel any uploads which are in progress.
!GC
Issues a System.gc() command and then displays "GC success."
!LOADMASTER
Reload the resource master
!PURGE
Same as !FLUSH but clearing document related caches and calling System.gc();
!PURGE2
Same as !PURGE but also doing a !FLUSHTHREAD and !FLUSHMAPSET before the System.gc();
!REQ
Shows current fox requests
!SIGNSTART
Start the digital signing job
!SIGNSTOP
Stop the digital signing job
!REPORTINGSTART
Start the jasper reporting job
!REPORTINGSTOP
Stop the jasper reporting job
!SECURITY[/DEFAULT]
Make the boot security file with optional generation of RSA key for you
!SETCOUNT/(baseline|disable|enable|full|lax|reset|resetbaseline)
Command that can be issued to change behaviour of object counting
!SETINTERNALPASSWORD
Change the internal password until the engine is restarted
!SHOWTRACK/(Integer)
Shows track data with a given history offset.
!SHOWMASTER
Shows the resource master currently in use
!SHOWGROUP
Shows the server group with the app that docgen processing will use
!TRACK/(Integer|(++|--|+|-)(FULL|XPATH|SQL))
If an integer is passed it'll set the track history size to that, else it'll set the mode. ++|-- changes global track modes and sets it locally too, +|- sets/unsets modes.
